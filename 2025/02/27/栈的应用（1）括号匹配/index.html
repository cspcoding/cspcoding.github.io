一、单种括号的简单匹配检验

1.1 P1739

1.1.1 原题链接

　　P1739 表达式括号匹配

1.1.2 分析

　　本题只需要分析左右小括号是否匹配，主要是看左括号的个数是否等于右括号的个数，且在从左往右扫描统计的过程中，随时要保证左括号的个数大于等于右括号的个数。本题并没有做其他更为复杂的小括号匹配操作（例如本题不考虑形如a(b-)c这种错误情况）。可以设计一个计数器cnt来统计左括号的个数，遇到左括号就cnt++，遇到右括号就cnt--且查看是否出现cnt<0，一旦出现则视为不匹配而立即结束，若扫描结束后cnt不为０也视为不匹配。该题很简单，入门级。

1.2 P3056 [USACO12NOV] Clumsy Cows S

1.2.1 原题链接

　　P3056 [USACO12NOV] Clumsy Cows S

　　大致题意：给出一个偶数长度的括号序列（字符串形式），问最少修改多少个括号可以使其平衡，并且左右括号要正确匹配。

1.2.2 分析　

　　因为要求左右括号必须正确匹配，所以并不是计算个数那么简单了。每个括号的位置不同，答案也不一样，无法用什么数学方法来搞定，也肯定不能排序（会打乱括号的顺序，从而使得答案发生变化），所以只能去模拟。

　　在模拟的时候，第一反应就是使用一个真实的栈（不论手写或者用STL的stack）去模拟左括号的进和出（“用两个栈分别存放左右括号”的另类做法是不可行的，会发现无法使用“出栈”操作）。如果输入的串是一个正确匹配的字符串，那么这种做法是可行的。但是根据题意，输入的字符串可能并不是一个正确匹配的字符串，模拟过程中可能会存在“需要出栈（串遇到右括号，需要把当前栈顶的左括号出栈）却没有数据可出栈（当前栈已为空）”的错误情况（例如串内容为“\(()))\)”）。当遇到这种情况，我们需要把当前的“)”视为"("并把"("进栈，这种修改需要记录次数，修改后进栈的"("应可以和后续的括号产生匹配。

　　用真实的栈来模拟，主要通过是否存在上述的红色文字来判断和统计需要修改的次数。其实仔细思考可以发现，要想统计出现红色字体的次数，不一定非要用真实的栈才能做到，我们可以用更为简单的方式，同样也能达到目的，那就是不用栈，在扫描串的同时只需要设计一个计数器leftsum去统计尚未配对的"("的个数即可。当从左往右扫描时，若遇到串的"("就leftsum++;　；若遇到串的")"就尝试去做leftsum--，若减之前发现leftsum==0，说明之前所有的字符都已经匹配成功，没有多余的"("可和当前的")"来匹配，产生错误，此时应把处理策略改为leftsum=1（左侧的所有符号均已配对成功，需要把新入的")"改为"("，以备和后续可能出现的")"匹配），同时ans++;。

　　如果输入的串是一个正确匹配的括号串，那么这个计数器的值就会不断地在０和一些正数中变化，不会为负数。需要注意一点的是，这个策略是否能处理所有的出错情况呢？这个时候正确的做法是需要在草稿纸上尝试去精心枚举所有出错的具有代表性的样例，看看当前的策略是否能够全部处理好这些样例。因为配对的括号序列会让计数器的值恢复原值，对我们枚举错误样例并无贡献，反而会成为干扰项。所以建议同学们在草稿纸上写错误样例的时候，就不要去写能产生配对的括号序列了，例如单个的"()"或者"((()))"等。下面是所有无法匹配的括号序列（长度必为偶数，想想为什么？）：

序号 原串内容 ')'修改次数 原串的)个数 原串的(个数 leftsum值结果 修改后的结果 答案
(1) )))) 2 ４ ０ leftsum=0 ()() ans=2
(2) (((((( 0 ０ ６ leftsum=6 ((())) ans=3
(3) )))(((((\(\quad奇数个右括号\) 2 ３ ５ leftsum=6 ()((())) ans=5
(4) ))))((((((\(\ \ \ 偶数个右括号\) 2 ４ ６ leftsum=6 ()() ((())) ans=5

可以发现，上面的后三种情况中leftsum != 0，这是因为左括号太多，没有足够的右括号与之匹配。可以按照情况２）来处理即可，即在扫描结束后，把多余的“(”改成“)”即可，即有ans= ans + lefusum/2。上面所有的橙色文字即为策略。

二、多种括号的简单匹配检验

2.1 P10472 括号画家

2.1.1 原题链接

　　P10472 括号画家

2.1.2 分析

　　题目要求录入一个混乱的只带有多种括号的字符串，要求检查并计算出其中最长的“合法”（即其内的括号是正确匹配的）子串的长度值。

　　这种题怎么做呢？排序和贪心肯定是用不上的，顺序一乱就没法做了；右括号（不管是哪一种）肯定是就近匹配离它最近的左侧的左括号（种类要吻合），每个右括号都只有唯一的左括号与之匹配，并无多种匹配策略，所以贪心和深搜也用不上。\(dp\)比较复杂，可能适合本题。模拟似乎也可以试试。思考一下之后觉得还是模拟起来可能比\(dp\)简单点，故先尝试模拟。

2.1.2.1 方法一分析：模拟　（时间复杂度\(O(n)\)，空间复杂度\(O(n)\)）

　　在从左往右扫描每个字符的过程中，如果遇到的是左括号，不管是哪一种，都先只记录下来。如果遇到的是右括号，则开始尝试去和事先记录的左括号内容匹配：若匹配失败，需要丢弃之前的所有计算成果，重新开始剩余子串的匹配；若匹配成功一对左右括号，则同步记录当前连续匹配成功的子串长度并刷新最大匹配长度。因为在右括号尝试去匹配左括号的时候，肯定会就近匹配，这就给我们用栈去模拟过程带来了机会，因为栈是先进后出的结构，最近的数据在栈顶位置。故而，我们可以设定一个栈st，专门用来保存需记录的左括号（为何没必要存右括号，想一下呢？）。在从左往右扫描的过程中，匹配策略如下：

　　1）若遇到左括号，一律入栈；

　　2）若遇到右括号，并且与当前栈顶符号正好是一对，则把这两个括号都标记为“匹配成功”，弹出栈顶的左括号（其已匹配，无需再存）；若与当前栈顶符号并不是一对，则说明当前字符已经匹配失败，之前存于栈中的所有左括号已无需再匹配（已经失败了，无用了），均应出栈，把栈清空，以备右侧的串内容重新开始。

　　我们已经解决了匹配的方法，但如何求出最长的连续匹配长度呢？一个最简单（不排除存在其他做法，但该做法最简单易懂）的做法就是设置一个标记数组，记录每个位置上的字符是否匹配成功。至于最长的连续匹配长度？那就是该数组中连续的最长的值为true的区间长度而已。这个很容易搞定。

　　上面讲述了模拟的方法，下面来看看\(dp\)的方法。

2.1.2.2 方法二分析：dp　（时间复杂度\(O(n)\)，空间复杂度\(O(n)\)）

　　要想用\(dp\)的方法，就必须要满足动态规划的两个要求（详情请见这里）：局部最优化原理和无后效性。从本题来看，每个右括号都会尝试只从最靠近它的未匹配的左括号去做匹配，这是唯一考虑的匹配对象，一旦不匹配则视为失败。当前dp[i]的决策细节、数组值不受后续数据的影响，且后续的决策与当前决策的细节无关（只和当前的dp[i]值有关）。所以理论上来说可以用dp来做。

１）状态数组定义

　　dp[i]表示以s[i]结尾的最长的括号匹配的连续子串的长度值

２）所求目标

　　\(max(dp[1..n])\)（假设数据从１号位开始存储）

３）状态转移方程

　　转移方程比较复杂，不容易凭空想出。此时需要同学们拿出纸笔，在草稿纸上写出多个具备代表性的数据，研究对应的dp[]值的规律，总结出来转移方程。例图如下：

在上图中尝试获取dp[i]的值，可以发现：

１）如果s[i]是左括号（不管是哪一类的左括号），则必不可能与其左侧的任何符号匹配，所以有dp[i]=0；

２）如果s[i]是右括号，则开始尝试匹配，具体的过程如下：

　　ａ）首先，要找到应该与s[i]进行匹配考察的是哪个字符。从上图的４个例子可以看出，在s[i]左侧可能存在已匹配成功的子串（绿色）或为空。往左跨过绿色的字符子串后的第１个字符就是和s[i]进行匹配考察的字符。该跨多少个字符呢？正好就是绿色子串的长度，也是dp[i-1]的值。例如在图（ｄ）中，与s[i]对应的需进行匹配考察的字符是\(\textcolor[RGB]{255,0,0}{s[\ i-1\ -\ dp[i-1]\ ]}= s[i-1-4] = s[i-5]\)；与s[i-1]对应的需进行匹配考察的字符是\(s[(i-1) -1 - dp[(i-1)-1]]=s[i-2-0] = s[i-2]\)。

　　ｂ）其次，在找到匹配考察的对象字符之后，开始匹配考察。

　　　　＃）若不匹配，则dp[i]=0

　　　　＃）若匹配，以图（ｄ）为例来说，则有“匹配的最大长度＝红色字符个数＋绿串长度＋蓝串长度”，一般化后有\(dp[i] = 2\ +\ dp[i-1]\ +\ dp[\ i-2\ -dp[i-1]\ ]\)。这就是本题的状态转移方程。

４）循环边界和顺序

　　ｉ从左往右，从２到ｎ

５）特例和初始化值

　　dp[0]= 0; dp[1]=0; 要想匹配成功，至少要２个字符，所以空串和单字符串都不存在成对的括号。

3.1 P3015 [USACO11FEB] Best Parenthesis S

3.1.1 原题链接

　　P3015 [USACO11FEB] Best Parenthesis S

3.1.2 分析

　　和P10472一样，着重在模拟和ｄｐ这两个方向上进行研究。下面先看模拟的方法。

3.1.2.1 方法1：模拟

　　我们在草稿纸上尝试写一些数据，看看能不能从中找出一些规律，以便于简化计算过程。

序号 串样例 计算过程 结算结果
(1) \((\ (\ \textcolor[RGB]{255,180,0})\ (\ \textcolor[RGB]{255,180,0})\ )\) (1+1)*2 2+2=4
(2) \((\ (\ \textcolor[RGB]{255,180,0})\ )\) 1*2 2
(3) \((\ \textcolor[RGB]{255,180,0})\ \ (\ (\ \textcolor[RGB]{255,180,0})\ )\) 1+1*2 1+2=3
(4) \((\ (\ \textcolor[RGB]{255,180,0})\ )\ \ (\ \textcolor[RGB]{255,180,0})\ \ (\ (\ \textcolor[RGB]{255,180,0})\ (\ \textcolor[RGB]{255,180,0})\ )\) 1*2+1+(1+1)*2 2+1+2+2=7
(5) \((\ \textcolor[RGB]{255,180,0})\ \ (\ (\ (\ (\ \textcolor[RGB]{255,180,0})\ (\ \textcolor[RGB]{255,180,0})\ (\ \textcolor[RGB]{255,180,0})\ )\ )\ )\) 1+(1+1+1)*2*2*2 1+(1+1+1)*8=1+8+8+8=25
(6) \((\ \textcolor[RGB]{255,180,0})\) 1 1

　　可以看出，“计算过程”中的每个“1”都和“串样例”中的黄色右括号相对应。而“计算过程”中的每个“*2”，都和“串样例”中的白色右括号相对应。并且，在计算过程中还需要注意层次的概念，同层的数据之间是“相加”的关系，初始时层次值level=0。每次遇到白色左括号就进入更深层次，有level++；每次遇到右括号的时候需要区别对待：1）若s[i]为黄色右括号（即s[i]==')' && s[i-1]=='('），则有level--, ans=ans + pow(2, level)。因为本题需要取模运算，所以每次累加了ans之后别忘了及时取模，同时pow(2,level)也需要改写，加入取模运算，否则会爆long long。2）若s[i]为白色右括号，则只有level--即可。

　　本题的破题关键在于：１）要在草稿纸上耐心写样例，写计算过程，不能心急，不要跳步骤或过度省略，使得不方便总结出规律；２）要把计算过程进行拆解，就会发现答案其实是多个“２的若干次方”的累加和值。进一步可以发现其指数和所在层次有关。这样题就变得很简单了。

3.1.2.2 方法２：ｄｐ

　　若从ｄｐ的角度来看，本题实际上是满足“局部最优化原理”和“无后效性”这两个条件的。设dp[i]表示串s[1..n]的得分。若s[i]=='('，此时并不会产生新的分数，所以只需要dp[i]=dp[i-1]即可。若s[i]==')'，这个时候必然会产生分数的变化：若为"()"的情况(s[i-1]=='(')，则有dp[i]=dp[i-1]+1；若为"))"的情况（s[i-1]==')'），则意味着需要把当前层的得分翻倍（此时的得分并不准，因为尚未扫描结束，此时的串并不是完整匹配的一个串），然后继续往右扫描，直到最后一个')'被扫描后（此时的串是完整匹配的），翻倍的次数就准确了。

表１ １ ２ ３ ４ ５ ６ 7 8
s[1:n] ( ) ( ( ) ( ) )
dp[1:n] 0 1 1 1 2 2 3 1+(3-1)*2=5
表２ １ ２ ３ ４ ５ ６ 7 8
s[1:n] ( ) ( ( ( ) ) )
dp[1:n] 0 1 1 1 1 2 1+(2-1)*2=3 \(\qquad\)🐱‍🐉
表３ １ ２ ３ ４ ５ ６ 7 8
s[1:n] ( ) ( ( ( ) ) )
dp[1:n] 0 1 1 1 1 2 3 1+(3-1)*2=5

🐱‍🐉：这个３并不是s[1:8]的得分，因为s[1:7]并不是一个正确的串，只有s[1:2]和s[1:8]才是正确匹配的字符串，对应的dp[2]和dp[8]的值才是正确的。

　　以表３为例，可以看出有一个问题需要解决，那就是：当计算dp[8]时，我如何去找红色１的下标位置，从而获得这个数据？我们知道s[8]和s[3]是配对的，而dp[3]正是我们要找的红１。我们可以借助一个栈结构来保存所有左括号在s[]中的下标值（遇到左括号就进栈，遇到右括号就出栈），从而帮助我们找到与每个右括号匹配的左括号的下标。这样就能轻松获得与８对应的３，从而获得dp[3]的值１了～～搞定！即有dp[i]= dp[ st[top] ] + ( dp[i-1] - dp[ st[top] ] ) * 2 ;。注意需要及时取模。
