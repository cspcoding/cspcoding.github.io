单调栈入门&找附近更大／小值

发表于2025-02-20|更新于2025-02-27|STL栈
|阅读量:

一、基础概念与分类

1.1 定义

　　单调栈是一种栈结构，通过进栈和出栈操作，维护让栈内元素始终保持单调递增或递减的顺序。根据数值大小的方向不同分为：

  • 单调递增栈：从栈顶到栈底，元素依次单调递增

  • 单调递减栈：从栈顶到栈底，元素依次单调递减

　　小细节：

　　１）本站在书写数组时，从左往右下标依次增加；在书写单调栈时也是从左往右依次进栈，故右侧为栈顶。在判断一个栈是递增或递减栈时，应从右侧往左侧看。例如单调栈[10,20,30]是递减栈，而单调栈[30,20,10]是递增栈。

　　２）新元素进栈后，栈顶元素即为nums[0..i]范围内的单调最优值。在不同的题型中，该值会具备不同的应用意义。

　　３）为区分数据值相同的情况，使得表达更精准，通常在编码时，入栈的都是nums[]中的下标值i，而不是nums[i]。但是为了讲述方便，本帖后面所有文字中，凡是提到“栈内值”或者“栈顶值”等，指的都是根据栈中存储的下标值i所对应的nums[i]中的值。并且，所谓的递增栈或者递减栈，指的都是nums[]值的递增或递减，而不是nums[]下标值的递增或递减。例如对nums[1..5]=[５，30,20,10,40,50]进行处理时，假如现在栈中存储的是前三个数，我会说“当前栈内存储的是[30,20,10]，它是一个递增栈”，但同学们应该要明白，实际栈中存储的是[1,2,3]。

1.2 题型分类

解决一类边界查找问题，主要分为以下几类：

  • 找附近更大／更小的数

    • 找附近更大数，[200, 100, 30, 10] vs 60，维护一个递增栈，当满足栈顶<新元素时，连续出栈10和30，变成[200, 100]，出栈的30和10的“右侧更大数”就是该新元素60（出栈前标注）；现在的栈顶元素100即为新元素60的“左侧更大数”（进栈前标注）。栈变为[200, 100, 60]
    • 找附近更小数，[10, 30, 100, 200] vs 60，维护一个递减栈，当满足栈顶>新元素时，连续出栈200和100，变成[10, 30]，出栈的200和100的“右侧更小数”就是该新元素60（出栈前标注）；现在的栈顶元素30即为新元素60的“左侧更小数”（进栈前标注）。栈变为[10, 30, 60]

  • 柱状图中的最大矩形面积

  • 按字典序构造子序列

注意：“右侧更小”的意思是：寻找比nums[i]更小且位于其右侧的第一个数，例如nums[]=[10,20,5]，则10右侧更小的数是5。但“$右侧更小 \neq 左侧更大$”。上例中5的左侧更大的数是20，并不是10。

二、工作原理与算法流程

2.1 操作示例

　　以记录每个数往左数第一个比自己更小的数（简称“左侧更小”问题）为例。

　　nums数组 假设为[50, 30, 70, 40, 60]，要求在栈内，栈顶元素左侧的数比栈顶更小。根据单调栈关于递增／递减的定义规则，从栈顶到栈底依次递减，故应定义单调递减栈。其详细操作过程如下：

  • i=0,nums[i]=50：栈空 　　　→ 50入栈：[50]（右侧为顶）　　　→　50左侧第一个小于50的数是“虚无”
  • i=1,nums[i]=30：栈顶50>30 → 50出栈，30入栈 ：[30]　　　　　→　30左侧第一个小于30的数是“虚无”
  • i=2,nums[i]=70：栈顶30<70 → 无需出栈，70入栈 ：[30, 70]　　→　70左侧第一个小于70的数是30
  • i=3,nums[i]=40：栈顶70>40 → 70出栈，40入栈 ：[30, 40]　　　→　40左侧第一个小于40的数是30
  • i=4,nums[i]=60：栈顶40<60 → 无需出栈，60入栈 ：[30,40,60]　→　60左侧第一个小于60的数是40

注意：

1）进栈后，若单调栈内只有1个数据时，可假设左侧为“虚无（以特殊值来代替（例如INF或者-1等）或进行特判）”，则建立“虚无”和该数据之间的关联；

２）不管是单调递增栈，或者是单调递减栈，新元素都会进栈，并且栈内元素值的递增／递减曲线应尽可能平缓。也可以从这个角度来理解上面的样例中为何70会被出栈。从30到70增长太陡了，肯定没有从30到40到60那么“平缓”。

3.2 P2866

3.2.1 原题链接

　　P2866 [USACO06NOV] Bad Hair Day S

3.2.2 分析

　　单调栈的入门级裸题之一。题目表述有点智障，从左往右编号依次为１到N，左为前，右为后（题目这里说反了）。

　　以nums[]=[ 10, 3, 7, 4, 12, 2] 为例，统计每个数有多少个比自己小的数（遇到比自己大的就停止统计，实际上就是“找右侧更大数’）。题目文字说得不太清楚，但通过样例１可以更为准确地理解题意。该样例答案即为 3个(10) + 0个(3) + 1个(7) + 0个(4) + 1个(12) + 0个(2) = 5个 。解释如下：

　　1. 身高为10的牛能往后看到身高为3、7、4的奶牛，但会被身高为12的奶牛挡住，从而无法看到身高为2的奶牛

　　2. 身高为3的牛往后看不到任何身高比自己矮的奶牛

　　3. 身高为7的牛能往后看到身高为4的奶牛，但会被身高为12的奶牛挡住，从而无法看到身高为2的奶牛

　　4. 身高为4的牛往后看不到任何身高比自己矮的奶牛

　　5. 身高为12的牛能往后看到身高为2的奶牛

　　6. 身高为2的牛往后看不到任何身高比自己矮的奶牛

　　从左往右录入每个数，在处理当前数据时，右侧的数据还没录入，是未知的。这为统计数据带来不便。可以转换一下思路：

　　“能看见３个数” 是等效于“３个数能被看见”的。所以“10能被后面的3、7、4看到”，是等效于“3能看到10，7能看到10，4能看到10”。我们无法在处理10时当即知道这个“３次”，但我们可以在处理3、7、4的时候分别在这三个数上各标注１次。这３次标注并非一次性完成的，而是在处理各个数时，分别去做的。

　　从左往右录入数据时，可以用栈来维护一个子序列，如果栈顶元素　>　新进元素，即栈顶元素能看到新进元素，那栈顶元素左侧的更大的那些元素自然更能看到新进元素，无需再去比较。所以维护这样的单调递增栈是有效率的。而如果栈顶元素　<= 新进元素，说明栈顶元素被新进元素挡住了，不管后续的元素值或大或小，该栈顶元素都看不到它们了，已无存储价值，可以弹出丢弃了。所以，这个子序列设计成递增栈（从右往左值递增）是可行的。

栈s内数据[左底…右顶] 策略 次数
[10] 栈空，10进栈 ＋0
[10,3] 栈顶元素（栈内最小值）10　>　新元素3，栈内所有元素都能看到3，3进栈 +s.size()=+1
[10,7] 栈顶元素（栈内最小值）3<新元素7，3被7挡住，再也不可能看到后续数，3已对后续数据无任何答案贡献，出栈3；新栈顶元素（栈内最小值）10>新元素7，栈内所有元素都能看到7，7进栈（你看这效果，是不是很像“单调递增栈”？） +s.size()=+1
[10,7,4] 栈顶元素（栈内最小值）7　>　新元素4，栈内所有元素都能看到4，4进栈 +s.size()=+2
[12] 栈内3个数都被12挡住，都不可能再看到12右侧的数，全废，全出栈。12进栈 +0
[12,2] 栈顶元素（栈内最小值）12　>　新元素2，栈内所有元素都能看到2，2进栈 +s.size()=+1
累计＋５

　　从上表可以看到，只要我们维护一个单调递增栈，每次在退栈完毕后累加当前栈长度值，即可搞定。

　　同学们可以思考：为何这里不是单调递减栈呢？

　　同学们可以以nums[]=[ 10, 3, 7, 4, 12, 2]为例，依次凑单调递减栈，其过程为[10] => [3] => [3,7] =>......。当遍历到3时，因为构造的是单调递减栈，所以10需要出栈，但其实10不应该出栈。因为后面的7和4还没遍历到，都比10小的。它们都能被10看到，或者说它们都能看到左侧的10。现在一旦把10出栈丢弃，后面再也无法找到它，无法再做比较，这里就产生了信息丢失。那为什么递增栈就可以成功呢？当从[10,3]=>[10,7]时也发生了“把３出栈丢弃”的事，因为现在出现了7，比3大，根据题意，3右侧不可能再出现比3小的数了，即使7右侧还有2或者1之类的，也会被7挡住。所以针对3的统计已经结束了，可以丢弃了。

　　739. 每日温度

3.3.2 视频教程

3.3.2 分析

　　题意很好理解，是典型的＂求最近更大＂的题型，在找到最近更大值之后，本题需要计算这两个参与比较的数之间的间隔数，这个其实很简单，两者对应的下标值之差就是了。栈内存储的是下标值。这个题有两种做法，同学们都可以试试看，扩展一下思路。两种做法的难度是一样的。

（一）从左往右遍历和统计

i=0：１（０)进栈；

i=1：对于１来说，４就是１的右侧更大值，标记$ans[0]=1-0=1$，１（０）出栈，４（１）进栈

i=2：３（２）进栈；

i=3：对于３（２）来说，５就是３的右侧更大值，标记$ans[2]=3-2=1$，３（２）出栈；

　　 对于４（１）来说，５也是４的右侧更大值，标记$ans[1]=3-1=2$，４（１）出栈；　５（３）进栈

i=4：５（４）进栈（相等的数据仍需进栈，因为并未大于左侧的那个等值，左侧等值不能出栈）

i=5：２（５）进栈

i=6：对于２（５）来说，３就是２的右侧更大值，标记$ans[5]=6-5=1$，２（５）出栈；３（６）进栈

i=7：对于３（６）来说，６就是３的右侧更大值，标记$ans[6]=7-6=1$，３（６）出栈；

　　 对于５（４）来说，６也是５的右侧更大值，标记$ans[4]=7-4=3$，５（４）出栈；

　　 对于５（３）来说，６也是５的右侧更大值，标记$ans[3]=7-3=4$，５（３）出栈；

$ans[7]$未作标记，仍为初值０，它右侧无数据，故而它无解。

　　可以看出，需要维护一个从栈顶到栈底依次递增的单调递增栈，在每个数据出栈前进行数据标记即可。

3.3 B3666

3.3.1 原题链接

　　B3666 求数列所有后缀最大值的位置

3.3.2 分析

　　题目文字非常绕，又是“后缀”，又是“最大值”，又是“下标”，再来个“所有下标的异或’，加上个逐渐变长的序列ａ，大部分小朋友可能无法读懂这个题。不过，看样例１倒是能慢慢理解准确。下面对样例１的nums[1..5]=[20,10,30,50,40]做个详细说明（为了方便突出展示栈内元素值的大小关系，在展示栈内容时给同学们展示的是nums[]的值，而不是其下标。实际上栈中存储的是下标，而不是nums[]值）：

１）i=1，子序列nums[1]=[20]。子序列中的后缀最大值只有第１个数20，栈内容为[20(1)]，输出$0\oplus1=1$；

２）i=2，子序列nums[1..2]=[20,10]。因为在子序列中，第１个数20比后面所有的数（当前只有10）更大），且第２个数10比后面所有的数（当前无）更大，所以子序列nums[1..2]中的后缀最大值有20和10，对应的下标分别为１和２，栈内容为[20(1),10(2)]，输出$1\oplus2=$1^2$=3$；

３）i=3，子序列nums[1..3]=[20,10,30]，因为在子序列中30最大且在末尾，根据题意，子序列中的后缀最大值只有第３个数30，20和10需要出栈，栈内容为[30(3)]，输出$0\oplus3=3$；

４）i=4，子序列nums[1..4]=[20,10,30,50]，因为在子序列中50最大且在末尾，根据题意，子序列中的后缀最大值只有第４个数50，30需要出栈，栈内容为[50(4)]，输出$0\oplus4=4$；

５）i=5，子序列nums[1..5]=[20,10,30,50,40]，因为在子序列中，第４个数50比后面所有的数（当前只有40）更大），且第５个数40比后面所有的数（当前无）更大，所以子序列nums[1..5]中的后缀最大值有50和40，对应的下标分别为４和５，栈内容为[50(4),40(5)]，输出$4\oplus5=$4^5$=1$；

　　到这里，相信每个同学都能准确理解题意了。那如何快速地找出上述中橙色文字的内容呢？这就要用到单调递增栈（从栈顶（右）到栈底（左），栈元素(下标)对应的nums[]值依次增大）了。为什么不是单调递减栈呢？同学们可以自己试试nums[1..5]=[2,1,3,5,4]这个样例，在栈内维护一个从左往右依次递增（即从栈顶到栈底元素值对应的nums[]值依次递减，即单调递减栈）的序列，你会发现你拿着这个序列很难实现本题的要求。

　　另外，如何对栈内所有元素（已是下标值）进行累计异或呢？可以单设一个变量ans，每次进栈i时就做ans=ans^i;，而每次出栈的时候就做ans=ans^s.top();（假设ans==x^a，根据异或操作的特点ans^a =(x^a)^a =x^(a^a) = x^0 =x，所以该操作相当于“在ans值中消除s.top()对其的影响”）。

　　每个录入的元素值的范围在$1\leq x_i \leq 2^{64}$，这个上限是很大的，只能用unsigned long long才能搞定。同时，因为ｎ的规模也达到了$1e6$的级别，所以只能用scanf/printf或者取消了同步后的cin/cout才行。如果用scanf/printf，对于unsigned long long类型，其对应的通配符是%llu。
