（一）不限宽度的区间的“最大的和值”问题

发表于2024-12-24|更新于2025-03-27|线性结构编程技巧最大子段和
|阅读量:

　　本帖是“最大子段和”系列问题的第1帖，主要研究不限宽度的区间的“最大的和值”和“最大的积值”这两个问题。

一、不限宽度的区间（子段）的"最大的和值"

1.1 原题链接

　　P1115 最大子段和　／　53 最大子数组和

1.2 分析

　　如果我们用 \(dp[i]\) 表示前 i 个元素中选择的子段内所有数的最大和值，\(a[]\)表示题目上对应的数组 ，那么所求目标即为\(dp[n]\)。但我们会发现，要根据\(dp[i-1]\)等来递推出\(dp[i]\)是很困难的，因为我们不确定\(a[i-1]\)在\(dp[i-1]\)的方案中是否被选中。如果\(a[i-1]\)曾被选中，那现在去计算\(dp[i]\)就比较简单了，对于\(a[i]\)无非有“选\(a[i]\)”或“不选\(a[i]\)”这两种方案，所以有\(dp[i]=max(dp[i-1]+a[i],\ \ dp[i-1])\)。但如果\(a[i-1]\)不曾被选中，那根据本题对“子段”的“下标连续性”的要求，就很难去推导了。

　　此路不通。不通的原因不是因为不该用\(dp\)，而是因为\(dp[i]\)的定义不对，因为违背了\(dp\)的定义并不满足「最优子结构」(即“问题的最优解包含子问题的最优解”。反过来说就是，我们可以通过子问题的最优解，推导出问题的最优解)。而在当前\(dp[]\)的这种定义下，当前位置的最优解未必是由前一个位置的最优解转移得到的。

　　正确的定义应该是用 \(dp[i]\) 表示"在选定\(a[i]\)的前提下，前 i 个元素中选择的子段内所有数的最大和值"。之所以正确，是因为在这种定义下，强制要求选定\(a[i]\)后，剔除掉了上面段落中所提到的“如果\(a[i-1]\)不曾被选中”的情况，使得\(dp\)的定义满足了“最优子结构”的要求。只不过在这样的定义下，所求目标不再是\(dp[n]\)，而变成了\(max(dp[1..n])\)。

　　这个题型，\(O(n)\)的做法有两种。

　　１）\(dp[i]=max(\ dp[i-1],\ \ 0)+a[i]\ )\qquad(i \in[1..n])\)

　　２）\(dp[i]=sum[i]-min(\ sum[1..i-1]\ )\qquad(\ i \in[1..n]\ )\)

　　第１种方法，在本站《前缀和　二维前缀和》有非常详细的讲述，请点击这里直达讲述位置，内含视频教学和AC代码，此处不再提。下面重点讲解第２种方法的得来。　

　　在上图中，我们以\(i=3\)为例来进行讲解。我们知道\(dp[i]\)的含义是“在选定\(a[i]\)的前提下，前\(i\)个元素中选择的子段内所有数的最大和值”，且用\(a[1..3]\)表示\(a[1]+a[2]+a[3]\)的值，则我们可以把\(dp[1]、dp[2]、dp[3]、dp[4]\)的定义演绎成下面的公式： \[ \begin{align} dp[1]&=max(\ a[1..1]\ ) \tag{1} \\ dp[2]&=max(\ a[1..2],\ \ a[2..2]\ ) \tag{2} \\ dp[3]&=max(\ a[1..3],\ \ a[2..3],\ \ a[3..3]\ ) \tag{3} \\ dp[4]&=max(\ a[1..4],\ \ a[2..4],\ \ a[3..4],\ \ a[4..4]\ ) \tag{4} \\ \end{align} \] 　　由（4）推出更加普遍的公式如下： \[ dp[i]=max(\ a[1..i],\ \ a[2..i],\ \ ......,\ \ a[i-1..i],\ \ a[i..i]\ ) \tag{5} \] 　　假设用\(sum[i]\)表示\(a[1..i]\)，即\(sum[i]=a[1]+a[2]+...+a[i]\)。用前缀和思想，可以把（５）式进一步演变如下： \[ \begin{align} dp[i]&=max(\ \qquad\textcolor[RGB]{50,200,0}{a[1..i]},\qquad\qquad\quad \textcolor[RGB]{0,150,255}{a[2..i]},\ \ \ \quad\quad ......,\ \ \qquad\textcolor[RGB]{255,0,0}{a[i-1..i]},\ \ \qquad\qquad\quad\textcolor[RGB]{255,225,0}{a[i..i]}\qquad\qquad) \tag{}\\ &=max(\ \textcolor[RGB]{50,200,0}{sum[i]-sum[0]},\ \ \textcolor[RGB]{0,150,255}{sum[i]-sum[1]},\ \ ......\ \ ,\ \ \textcolor[RGB]{255,0,0}{sum[i]-sum[i-2]},\ \ \textcolor[RGB]{255,225,0}{sum[i]-sum[i-1]}\ ) \tag{}\\ &=sum[i]-min(\ sum[0],\ sum[1],\ sum[2],\ ......\ ,\ sum[i-1]\ ) \tag{6} \end{align} \] 　　从公式（６）中可以看到，我们需要在对应的前缀和数组sum[]中去求多个前缀和的最小值。在本题中因为没有限制子段的区间宽度，所以公式（６）比较简单。但如果要求区间的宽度受限制（\(==m、\leq m、\geq m\)）等，公式（６）会变得稍微复杂一点，就需要用到单调队列来解决了。要实现方法２中的公式（６），可以有两种版本。

版本１）：用单变量\(mmin\)来保存\(sum[1..i-1]\)中各和值的最小值。

　　为了给后续帖展示复杂区间宽度限制，此处也给同学们展示做法２的过程。

版本２）：用单调队列来处理\(sum[1..i-1]\)中各和值的最小值。需要注意的是，此处是求滑窗区间\(sum[0..i-1]\)的最小值，基础数组是\(sum[]\)；而且在第\(i\)轮处理的是数据是\(sum[i-1]\)，进队的下标应该是\(i-1\)。

二、不限宽度的区间（子段）的“最大的积值”

2.1 原题链接

　　１５２　乘积最大子数组（leetcode）

2.2 分析

　　如果我们用 \(dpmax[i]\) 表示以第 i 个元素结尾的子段内所有数的最大乘积值，\(a[]\)表示题目上对应的数组 \(nums[]\)，那么根据“P1115 最大子段和”的经验，我们很容易推导出这样的状态转移方程： \[ dpmax[i]=max(\ a[i],\quad dpmax[i-1]*a[i]\ )\qquad(i \in[1..n]) \tag{1} \] 　　它表示以第 i 个元素结尾的乘积最大子数组的乘积，可以考虑 \(a[i]\) 加入前面的 \(dpmax[i-1]\) 对应的一段，或者单独成为一段，且在两种情况下取最大值。最后把 \(max(dpmax[1..n])\)作为答案。

　　可是在这里，这样做是错误的（需要孩子们认真仔细地在草稿纸上枚举多个样例，就必然能发现这一点）。为什么呢？

　　因为这里的定义并不满足「最优子结构」(即“问题的最优解包含子问题的最优解”。反过来说就是，我们可以通过子问题的最优解，推导出问题的最优解)。具体地讲，如果 \(a[]=\{5,\ 6,\ −3,\ 4,\ −3\}\)，那么此时\(dpmax[]\)对应的序列是 \(\{\ 5,\ 30,\ −3,\ 4,\ −3\ \}\)，按照前面的算法我们得到的答案为 30（即\(5\times6\)），而实际上答案应该是全体数字的乘积。我们来想一想问题出在哪里呢？问题出在最后一个 −3 所对应的\(dpmax[5]\)的值既不是 −3，也不是 \(4×(−3)\)，而是 \(5×6×(−3)×4×(−3)\)。所以我们得到了一个结论：在这种定义下，当前位置的最优解未必是由前一个或前几个位置的最优解转移得到。

　　我们可以根据正负性进行分类讨论。

　　１）如果\(a[i] <0\)，那么我们希望\(dpmax[i-1]<0\)，这样就可以负负得正，并且我们希望\(dpmax[i-1]\)尽可能"负得更多"(即尽可能小)。

　　２）如果\(a[i]>0\)，我们更希望\(dpmax[i-1]>0\)，并且希望\(dpmax[i-1]\)尽可能地大。

　　于是这里我们可以再维护一个 \(dpmin[i]\)，它表示以第 i 个元素结尾的乘积最小子数组的乘积，那么我们可以得到这样的动态规划转移方程： \[ \begin{align} dpmax[i]&=max(dpmax[i-1]*a[i],\quad dpmin[i-1]*a[i],\quad a[i]) \qquad(i \in[2..n]) \tag{2}\\ dpmin[i]&=min(dpmax[i-1]*a[i],\quad dpmin[i-1]*a[i],\quad a[i]) \qquad(i \in[2..n])\tag{3} \end{align} \] 　　特例值很好分析，\(dpmax[1]=dpmin[1]=a[1];\)

　　仔细分析公式（２）和（３），可以发现，\(dpmax[]\)和\(dpmin[]\)不必开数组，只需要用两个变量\(dpmax\)和\(dpmin\)就能搞定了。
