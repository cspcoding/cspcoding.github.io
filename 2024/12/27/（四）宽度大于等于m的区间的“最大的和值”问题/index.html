（四）宽度大于等于m的区间的“最大的和值”问题

发表于2024-12-27|更新于2025-01-01|线性结构编程技巧最大子段和
|阅读量:

　　本帖是“最大子段和”系列问题的第4帖，主要研究宽度大于等于m的区间的“最大的和值”这个问题。

一. 理论教学

　　之前我们练习过“区间宽度\(\leq m\)”的情况，用单调队列来解决了问题。已经记不太清楚的同学请看这里。看到当前这个题型，第一反应就是“能不能用之前解决\(\leq m\)的公式推导过程来解决本题型呢？”

　　可以试试看！

　　假设\(m=3\)，原始数组为\(a[1..n]\)，对应的前缀和数组为\(s[1..n]\)，\(dpmax[i]\)表示"在选定\(a[i]\)的前提下，子段内所有数的最大和值"，且满足子段宽度\(\geq m\)，而写法"\(a[3..5]\)"表示"\(a[3]+a[4]+a[5]\)"，则有 $$ \[\begin{align} dpmax[1]&=max(\ a[1..1]\ ) \qquad(区间宽度均不够3，无解)\tag{1} \\\\ dpmax[2]&=max(\ a[1..2], a[2..2]\ ) \qquad(区间宽度仍均不够3，无解)\tag{1} \\\\ dpmax[3]&=max(\ \ a[1..3]　\ \ ) \qquad(只有[1..3]这个区间满足:宽度\geq 3)\\ &=max(\ s[3]-s[0]\ ) \\ &=s3-min(\ s[0]\ ) \tag{3}\\\\ dpmax[4]&=max(\ \ a[1..4],\ \ \qquad\quad a[2..4]\quad ) \qquad (有[1..4]和[2..4]这２个区间满足：宽度\geq 3)\\ &=max(\ s[4]-s[0],\ \quad s[4]-s[1]\ ) \\ &=s[4]-min(\ s[0],\qquad\ s[1]\qquad\ )\tag{4}\\\\ dpmax[5]&=max(\ \ a[1..5],\ \ \qquad\quad a[2..5],\ \ \qquad \quad a[3..5]\ \ ) \\ &=max(\ s[5]-s[0],\ \quad s[5]-s[1],\ \quad s[5]-s[2]\ ) \\ &=s[5]-min(\ s[0],\quad \qquad s[1],\ \qquad\qquad s[2]\quad\ ) \tag{5}\\\\ dpmax[6]&=max(\ \ a[1..6],\ \ \qquad\quad a[2..6],\ \ \qquad \quad a[3..6]\ \ ,\ \ \quad \quad a[4..6]\ \ ) \\ &=max(\ s[6]-s[0],\ \quad s[6]-s[1],\ \quad s[6]-s[2]\ ,\ \quad s[6]-s[3]\ ) \\ &=s[6]-min(\ s[0],\quad \qquad s[1],\ \qquad\qquad s[2],\ \qquad\qquad s[3]\quad) \tag{6}\\\\ \end{align}\] \[ 则有一般性公式推导（注：从$\ i-m+1\ $到$\ i\ $，一共有$m$个数；从$1$到$i-m+1$，或者从$0$到$i-m$，都是有$i-m+1$个数）如下： \] \[\begin{align} dpmax[i]&=max(\ \ a[1..i],\ \ \qquad\quad a[2..i],\quad\ \ ......\ ,\ \qquad a[i-m..i],\ \qquad\ \ a[i-m+1..i]\ ) \\ &=max(\ s[i]-s[0],\ \quad s[i]-s[1],\quad ......\ ,\quad s[i]-s[i-m-1],\quad s[i]-s[i-m]\ \ ) \\ &=s[i]-\textcolor[RGB]{255,0,0}{min(\ s[0],\qquad s[1]\qquad,\ ......\ ,\qquad \quad s[i-m-1],\qquad s[i-m]\ \qquad\ ) }\qquad(i \in [m..n])\tag{6}\\\\ \end{align}\] $$ 　　从上面的公式（６）可以看出：

１）我们把一个求最大值的问题，变成了一个求最小值的问题。

２）原本是求\(a[]\)的“和值”，现在改成了求\(s[]\)的最值。

３）在第\(i\)轮，\(min\)操作的操作数的个数随着\(i\)的增大而增大，也就是说，本题型并不是固定滑窗求最值，却和“（一）不限宽度的区间的“最大的和值”问题”中的单调队列处理方法比较类似。所以不需出队（因为没有滑窗宽度限制），甚至也可以不用单调队列而直接用变量\(mmin\)就可搞定。

2.1.1 原题链接

　　P1404 平均数

2.1.2 分析

　　原题的题意是“给一个长度为 \(n\)的数列，我们需要找出该数列的一个子串，使得子串的平均数最大化，并且子串长度 \(\geq m\)”，要求输出最大的平均数。

　　我们可以初步分析一下：

　　１）要找一个子串，也就是说，要找一个子序列，或者说子段。

　　２）要想使得子段均值最大，也就意味着并不是子段越宽越好或者越窄越好。假设当前所选子段的均值为５，而新加入的数值为\(8>5\)，则势必会拉高均值，这是有益的；但如果新加入的数值为\(2 <5\)，则势必会拉低均值，是有害的。但有害的数据并不意味着就不能选，万一在２后面的下一个数是１００呢？所以，这并不是一个简单的贪心策略能搞定的，多半会和暴力枚举所涉及的算法有关。目前已经学过的暴力枚举的算法，除了简单模拟之外，有\(dfs\)、\(dp\)和二分答案。\(dfs\)复杂度太高，基本上放弃。

　　３）要求子段长度\(\geq m\)，这个条件如何利用呢？这个感觉和题型H：“借助单调队列来求子段宽度\(\geq m\)的最大的和值”很相似啊。但题型H是求和，而当前题型是求平均数，而且还要求最大的平均数。这里需要一个外力点拨。我们可以把原始数组元素都减去一个假定的均值\(avg\)，然后对数组再来做题型H，这样就解决了“子段长度\(\geq m\)”的问题。若求出的最大和值\(>0\)，说明\(avg\)太小了；若最大的和值\(<0\)，说明\(avg\)太大了。这听上去是不是很像“二分答案？的确是的。至此，已经完全分析清楚了。

　　例如：对于\(a[10]=\{2,5,7,9,6,1,6,7,4,1\},m=3\)，假设\(avg=5\)，则\(a_1^*[10]=\{-3,0,2,4,1,-4,1,2,-1,-4\}\)，对\(a_1^*[]\)求子段宽度\(\geq 3\)的最大子段和为\(2+4+1=7\)，这意味着什么呢？这说明这三个数对应的\(7,9,6\)原数的真实均值是大于当前的\(avg=5\)的，换句话说，当前的\(avg\)太小了，可以再大一点。

　　二分答案的代码模板就不多说了，若不清楚请复习本站帖"二分查找和二分答案.md"，点击这里。注意对check(x)函数的把握。

　　需要注意的是，本题并不是标准的二分答案，因为题目要求输出的是二分后的结果砍掉末尾后的精确值，而不是四舍五入后的结果。

　　鉴于上面的问题，所以需要把输出改为打印\(\ ri\ \)端值就可以了。
