多维费用背包问题

发表于2024-05-23|更新于2024-05-22|基础算法动态规划背包问题
|阅读量:

一、理论教学　　

1.1 模型引入

　　同学们还记得我们之前学过的普通01背包的模型及解法吧？如果不记得，请先复习回看，然后再看本帖。

　　之前的普通01背包模型中，背包总约束量YS1，每个物品的约束量ys1[]和收益value[]都给定，求在总约束量YS1之内的最大价值。这个题面有一个假设前提，那就是“只有一个背包”。那可不可以有两个或者多个不同类型的背包呢？当然可以有。下面就以二维为例，来讲解多维费用背包的解决方案。至于三维、四维等，解法思路是类似的。

　　二维费用背包的模型：

　　现在已知有两个属性的背包约束总量分别为YS1和YS2，每个物品在这两个属性上的约束量为ys1[]和ys2[]，该物品的收益值为value[]，求分别在这两个背包总容量限制之内的物品最大价值。(后续会有求最小值的模型，到时候再说。本帖都是求最大值的模型)

1.3 分析

　　对于二维费用背包（多维类似处理即可），只需要在遍历填写dp多维数组时，多加一个维度即可。当然，因为数组多了一个维度，所以总量更容易超过内存限度。往往在绝大多数情况下，只能使用滚动数组来处理多维费用背包的问题，否则会MLE。

1.3.3 时间复杂度分析

　　二维费用背包的时间复杂度都是\(O(n*YS1*YS2)\)，如果采用滚动状态数组的话，其空间复杂度为\(O(YS1*YS2)\)。

2.2.1 原题链接

　　P1910 L 国的战斗之间谍

2.2.2 分析

　　这个题也是一个几乎裸的二维费用背包题目，其解法代码和2.1节题几乎完全一致，可以作为复习和巩固。如果很熟了，可以不做本题。在本题中A是收益（价值），B和C都是成本。

　　求不超过背包总限制YS1（对应B）和YS2（对应C）的最大收益价值。\(n \leq 10^4,\ YS1 \leq 1*10^3,\ YS2 \leq 1*10^3\)。对于value[]没说其数值的取值范围，为了稳妥起见，最好是开long long。牢记：“十年OI一场空，不开long long见祖宗！”

2.3 P1855 榨取kkksc03

2.3.1 原题链接

　　P1855 榨取kkksc03

2.3.2 分析

　　这个题是一个二维费用背包题目，题面上明说了，在时间和金钱有限的情况下求达成愿望最多的个数。所以每个愿望所需的时间和金钱是成本，分别是ys1和ys2，而愿望个数是收益。那收益值是多少呢？很简单，纳入每个愿望后，达成1个愿望，所以收益值是1。所以本题不需要录入value[]哦！

　　本题数据规模较小，随便用标准状态数组或者滚动数组来做，都是可以的！
